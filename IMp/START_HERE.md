## ⚠️ What NOT To Do

❌ **Don't redesign everything from scratch**
- This is based on real user research
- Changes should be validated with users, not assumptions

❌ **Don't change the target users**
- Elderly living alone + caregivers is the right starting point
- These are the problems we've validated
- Expand after you've proven PMF with this segment

❌ **Don't over-build features in week 1**
- MVP should be bare-bones
- Talk to users before adding features
- Features are hypotheses, not certainties

❌ **Don't ignore the go-to-market**
- Building a great product isn't enough
- User research + partnerships = success
- See Sahaay_GTM_and_Research.md for launch plan

❌ **Don't launch without validating metrics**
- Week 4-8 is about measuring health impact
- If you don't see improvement in adherence, something's wrong
- Fix it before scaling

## 💡 Key Principles

1. **Real > Perfect**
   - 70% working with 10 real users beats 100% perfect with zero users
   - Build fast, learn from users

2. **Users First > Features**
   - Every decision should start with "What does the user need?"
   - Not "What's cool to build?"

3. **Health Impact > Growth**
   - Success is measured in adherence improvement, caregiver time saved
   - Not in "users downloaded" or "DAU"

4. **Sustainable > Explosive**
   - Profitable at 20K users is better than VC-dependent at 1M
   - Build a business, not a unicorn

5. **India First > Global**
   - Understand the India market deeply
   - Then expand to Southeast Asia, Middle East
   - Don't try to be global from day 1
